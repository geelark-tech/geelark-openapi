- GeeLark会将一些操作结果通过Webhook方式推送给开发者，设置Webhook URL后，开发者可以自行处理这些事件
  常用回调类型请参考文档：[回调类型](https://open.geelark.cn/api/callback-type "回调类型")

### 接口鉴权
用户自定义回调接口鉴权与请求GeeLark API一致

- 请求方式请设置为`POST`
- 请正确的接收`请求头信息`，token验证和key验证只需处理一种
- `Content-Type`设置为`application/json`

#### key验证所需请求头

- `appId` 团队AppId
- `traceId` 请求唯一id
- `ts` 毫秒级时间戳
- `nonce` 随机数
- `sign` 签名结果

#### key验证参数生成方式

- `traceId` 使用`Version 4 UUID`
- `nonce` 使用`traceId`的前6位
- `sign` 先拼接字符串`团队AppId`+`traceId`+`ts`+`nonce`+`团队AppKey`，使用字符串的`SHA256`十六进制大写摘要

#### token验证

请求时，会携带以下请求头

- `traceId` 请求唯一id
- `Authorization` 设置为`Bearer 从客户端获取到的token值`
