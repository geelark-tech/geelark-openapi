  常用回调类型请参考文档：[回调类型](https://open.geelark.cn/api/callback-type &quot;回调类型&quot;)

### 接口鉴权
用户自定义回调接口鉴权与请求GeeLark API一致

- 请求方式请设置为`POST`
- 请正确的接收`请求头信息`，token验证和key验证只需处理一种
- `Content-Type`设置为`application/json`

#### key验证所需请求头

- `appId` 团队AppId
- `traceId` 请求唯一id
- `ts` 毫秒级时间戳
- `nonce` 随机数
- `sign` 签名结果

#### key验证参数生成方式

- `traceId` 使用`Version 4 UUID`
- `nonce` 使用`traceId`的前6位
- `sign` 先拼接字符串`团队AppId`+`traceId`+`ts`+`nonce`+`团队AppKey`，使用字符串的`SHA256`十六进制大写摘要

#### token验证

请求时，会携带以下请求头

- `traceId` 请求唯一id
- `Authorization` 设置为`Bearer 从客户端获取到的token值`

### 开机回调接口接收参数
| 参数名 | 必选 | 类型 | 说明 | 示例 |
| --- | --- | --- | --- | --- |
| type | 是 | interger | 回调事件类型 | 1:开机事件 |
| id | 是 | string | 云手机id | 参考示例 |
| result | 是 | boolean | 执行结果 | true: 成功 false: 失败 |
| username | 是 | string | 操作者 | admin@geelark.com |
| ip | 是 | string | 操作者ip | 62.141.247.218 |
| eventTime | 是 | interger | 时间秒级时间戳 | 1766567054 |

#### 开机回调接口接收参数示例

```json
{
    &quot;type&quot;: 1,
    &quot;id&quot;: &quot;528086321789535232&quot;,
    &quot;result&quot;: true,
	&quot;username&quot; : &quot;admin@geelark.com&quot;,
	&quot;ip&quot; : &quot;62.141.247.218&quot;,
	&quot;eventTime&quot; : 1766567054
}
```
